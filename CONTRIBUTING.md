# Contributing

Thanks for helping improve OSS Maintainer Kit.

You do not need to be an experienced maintainer to contribute here. One of the main goals of this project is to make repository setup easier for people who are still learning GitHub and open source.

## Before opening a pull request

1. Keep changes focused.
2. Update docs when commands, file paths, or templates change.
3. Run `npm test`.

## Good contributions

- Simpler docs for first-time repo owners
- New starter presets with clear, beginner-friendly value
- Safer GitHub workflow defaults
- Better `AGENTS.md` examples for real repositories
- Documentation that shortens setup time without hiding important tradeoffs

## Please avoid

- Adding runtime dependencies without a strong reason
- Broad refactors that make the CLI harder to audit
- Example workflows that require unnecessary write access
- Copy changes that make the repo sound smarter while making it harder to understand
