# Roadmap

## Recently shipped

- standard label sync with a managed label manifest
- beginner, package, app, service, docs, and security-sensitive presets
- 8 public example repositories that show what each preset produces
- repo-specific social-preview cards for the full example repo family
- public case study derived from the `ShuleDocs` adoption work
- `check-docs` for local Markdown link and anchor validation

## Next up

- Add a dry-run diff mode so maintainers can preview generated changes.
- Add a release-note schema option for teams that want structured outputs.
- Add more presets for deployment targets and common stack conventions.
- Add lightweight scaffold helpers for common app docs like deploy, runbook, and architecture stubs.
